## Installation Methods

### Individual Components

Install any single component:

```bash
bunx shadcn@latest add https://tryelements.dev/r/{component-name}
```

### Component Suites

Install related components together:

```bash
# All Clerk authentication components
bunx shadcn@latest add https://tryelements.dev/r/clerk-auth-shadcn

# All theme switcher variants
bunx shadcn@latest add https://tryelements.dev/r/theme-switcher-collection

# Complete logo collection
bunx shadcn@latest add https://tryelements.dev/r/logos
```

### Provider Collections

Install all components for a specific provider:

```bash
# All Clerk components
bunx shadcn@latest add https://tryelements.dev/r/clerk-complete

# All theme components
bunx shadcn@latest add https://tryelements.dev/r/theme-complete
```

## Provider Setup Guides

### Clerk Authentication

Complete setup guide for Clerk components:

1. **Install Dependencies**

   ```bash
   npm install @clerk/nextjs @clerk/types
   ```

2. **Environment Variables**

   ```txt
   NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_...
   CLERK_SECRET_KEY=sk_test_...
   ```

3. **Provider Wrapper**

   ```tsx title="app/layout.tsx"
   import { ClerkProvider } from "@clerk/nextjs";

   export default function RootLayout({ children }) {
     return (
       <ClerkProvider>
         <html lang="en">
           <body>{children}</body>
         </html>
       </ClerkProvider>
     );
   }
   ```

4. **Middleware Configuration**
   The middleware is automatically installed with Clerk components.

### Theme Provider

Setup guide for theme components:

1. **Install Dependencies**

   ```bash
   npm install next-themes
   ```

2. **Provider Configuration**

   ```tsx title="app/layout.tsx"
   import { ThemeProvider } from "next-themes";

   export default function RootLayout({ children }) {
     return (
       <html lang="en" suppressHydrationWarning>
         <body>
           <ThemeProvider
             attribute="class"
             defaultTheme="system"
             enableSystem
             disableTransitionOnChange
           >
             {children}
           </ThemeProvider>
         </body>
       </html>
     );
   }
   ```

3. **CSS Variables**
   Ensure your CSS includes proper theme variables (automatically handled by shadcn/ui).

### Polar Sponsorship

Setup guide for sponsorship components:

1. **Install Dependencies**

   ```bash
   npm install @polar-sh/sdk
   ```

2. **Environment Variables**

   ```txt
   POLAR_ACCESS_TOKEN=polar_oat_...
   POLAR_ORGANIZATION_ID=your-org-id
   POLAR_SUCCESS_URL=https://yourdomain.com/sponsor/success
   ```

3. **API Configuration**
   Set up your sponsorship tiers in the Polar.sh dashboard.

## Component Architecture

### Design Principles

All Elements components follow these principles:

- **Theme Compatibility** - Dark/light theme support out of the box
- **Mobile First** - Responsive design for all screen sizes
- **Accessibility** - ARIA labels, keyboard navigation, screen reader support
- **Customization** - Easy styling with CSS classes and custom properties
- **Type Safety** - Full TypeScript support with proper type definitions

### File Structure

Components are organized with a consistent file structure:

```
components/elements/
├── clerk/
│   ├── sign-in.tsx          # Main component
│   ├── sign-up.tsx          # Related components
│   └── middleware.ts        # Supporting utilities
├── theme/
│   ├── switcher.tsx         # Core theme switcher
│   └── variants/            # Theme switcher variants
├── logos/
│   ├── apple.tsx            # Individual logo components
│   ├── google.tsx
│   └── index.ts             # Barrel exports
└── polar/
    └── sponsorship.tsx      # Sponsorship components
```

### Dependencies

Components are designed to have minimal dependencies:

- **Core Dependencies** - Only essential packages for functionality
- **Peer Dependencies** - shadcn/ui components and utilities
- **Optional Dependencies** - Provider-specific SDKs (Clerk, Polar, etc.)

## Usage Patterns

### Component Composition

Components are designed for easy composition:

```tsx
import { ClerkSignInElement } from "@/components/elements/clerk/sign-in";
import { ThemeSwitcher } from "@/components/elements/theme/switcher";

export function AuthLayout({ children }) {
  return (
    <div className="min-h-screen">
      <header className="flex justify-between p-4">
        <h1>My App</h1>
        <ThemeSwitcher />
      </header>

      <main className="flex items-center justify-center p-8">{children}</main>
    </div>
  );
}

export function SignInPage() {
  return (
    <AuthLayout>
      <ClerkSignInElement />
    </AuthLayout>
  );
}
```

### Custom Styling

All components support custom styling:

```tsx
<ClerkSignInElement className="w-full max-w-lg border-2 border-primary" />
<ThemeSwitcher className="bg-accent hover:bg-accent/80" />
<AppleLogo className="h-8 w-8 text-blue-600" />
```

### Provider Integration

Components integrate seamlessly with their respective providers:

```tsx
// Clerk integration
const { user, isSignedIn } = useUser();

// Theme integration
const { theme, setTheme } = useTheme();

// Custom logic with provider data
if (isSignedIn) {
  return <Dashboard user={user} theme={theme} />;
}
```
